# 文件级开发任务拆解清单（BOSS登录 + 滚动）

## 1. 目标与范围

- 目标：落地 `scrapy-playwright` 方案，完成 BOSS 场景的登录态维持、滚动加载采集、预览小窗联动。
- 范围：爬虫层、后端任务层、前端展示层、部署层四个模块。
- 约束：保留现有任务创建/状态流转能力，在此基础上增量改造。

---

## 2. 爬虫层任务（crawler）

## 2.1 依赖与设置

- [ ] `crawler/requirements.txt`
  - 新增 `scrapy-playwright` 依赖。
  - 按需补充 Playwright Python 依赖版本约束。
- [ ] `crawler/crawler/settings.py`
  - 增加 `DOWNLOADER_MIDDLEWARES` 与 Playwright 下载处理器配置。
  - 增加 `PLAYWRIGHT_BROWSER_TYPE`、并发上限、超时参数。
  - 关闭或调整与 Playwright 冲突的并发参数。

## 2.2 Spider重构（登录 + 滚动）

- [ ] `crawler/crawler/spiders/boss_jobs.py`
  - 默认核心请求走 `meta["playwright"] = True`。
  - 增加登录态检测逻辑（是否出现登录按钮/用户态元素）。
  - 增加登录流程回调（首期建议人工扫码登录保存会话态）。
  - 增加滚动循环逻辑（滚动、等待、计数、终止条件）。
  - 增加“连续无增长停止”策略，避免无限滚动。
  - 增加详情页解析流程，提取补充字段。
  - 增加阶段步骤上报（`check_login`、`scrolling`、`parse_detail` 等）。

## 2.3 Item与Pipeline

- [ ] `crawler/crawler/items.py`
  - 按最终数据模型补充字段（如步骤、抓取时间、来源标记）。
- [ ] `crawler/crawler/pipelines.py`
  - 保持 `source_url + crawl_task_id` 幂等写入。
  - 增加异常字段落库与基础数据校验。
  - 增加必要清洗（空值、字符串标准化、标签去重）。

- [ ] 详情页抓取成功率满足预期。

## 7.3 数据与状态验收

- [ ] Mongo 与 PostgreSQL 均有可用数据写入。

- [ ] 任务状态与前端展示一致。

## 9. 风险提示

- [ ] 风控触发导致登录态不稳定，需要人工介入预案。
- [ ] Playwright 资源占用高，必须先做并发限流。
- [ ] 滚动逻辑过快易触发反爬，需随机节奏与超时控制。
- [ ] 预览截图量增长快，必须配置过期清理策略。
